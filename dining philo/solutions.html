Prevention strategies

Synchronization policies for deadlock prevention

Three classical approaches used in operating systems to ensure safe and fair resource allocation: hierarchy, arbitration, and asymmetric protocols.

Resource hierarchy

Order the forks and always pick the lower-numbered fork first. Threads gain resources in a predetermined order so circular wait disappears.

Eliminates circular wait

Arbitrator / host

Introduce a coordinator who only seats four philosophers at a time. Limiting concurrent access breaks the hold-and-wait condition.

Removes hold-and-wait

Asymmetric strategy

Alternate fork pickup order by seat (left-right-left...). This tiny change prevents everyone from chasing the same resource simultaneously.

Breaks symmetry

Implementation notes

Mapping strategies to OS synchronization constructs

Each prevention technique directly corresponds to fundamental operating system primitives used for resource management and synchronization.

Hierarchy → lock ordering

Deadlocks vanish if every mutex is acquired in a global order. Many kernels enforce this for low-level locks.

Host → semaphore guard

Use a counting semaphore to cap how many threads can enter the critical region simultaneously.

Asymmetry → priority tweaks

Alternating lock acquisition order is a simple trick that fits well in teaching schedulers and fairness policies.
